# AIGovern Pro - 四大能力技术选型方案（一页纸）

```
┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤

         ↓              ↓               ↓              ↓
```

## 🎯 四大能力技术选型详解

### 1️⃣ **知识问答** (RAG + 向量库 + 大模型 + AGUI)

**核心流程**：用户提问 → 向量检索 → 知识片段召回 → 大模型增强生成 → AGUI展示

| 组件 | 技术方案 | 说明 |
|------|--------|------|
| **向量库** | Milvus | 海量知识库文档的向量索引和检索，毫秒级响应 |
| **大模型** | 豆包/通义千问 | 基于检索结果进行上下文理解和生成回答 |
| **RAG框架** | LangChain | 文档分块→向量化→检索→生成的完整流程 |
| **AGUI** | React ChatPanel | 右侧浮窗对话面板，展示回答+来源溯源+置信度 |
| **缓存** | Redis | 热门问题的缓存，加速响应 |

**技术指标**：
- 检索精度：Top-K相关度 > 0.85
- 端到端延迟：< 3秒
- 支持文档格式：PDF、Word、TXT、Markdown

---

### 2️⃣ **数据查询** (FunctionCalling + SQL生成/API调用 + 数仓)

**核心流程**：自然语言问题 → FunctionCalling决定查询方式 → SQL生成/API调用 → 图表展示

| 组件 | 技术方案 | 说明 |
|------|--------|------|
| **FunctionCalling** | 大模型内置能力 | 大模型理解用户意图，选择调用对应的函数/工具 |
| **SQL生成** | few-shot + 提示词 | 基于Schema定义和示例，生成准确的SQL查询 |
| **API调用** | RESTful接口 | 预定义的数据API，用于非SQL数据源 |
| **数仓** | PostgreSQL | 存储业务数据（订单、商品、用户等），支持OLAP查询 |
| **可视化** | Recharts | 自动选择合适的图表类型（柱状图/折线图/饼图等） |

**查询方式决策树**：
```
用户问题
  ├─ 聚合统计 → SQL生成 (SUM/AVG/COUNT)
  ├─ 多维分析 → SQL生成 (GROUP BY + JOIN)
  ├─ 实时数据 → API调用 (外部数据源)
  └─ 预计算结果 → Redis缓存 (秒级更新)
```

---

### 3️⃣ **智能操作** (A2UI + FunctionCalling + 前端Action协议)

**核心流程**：用户意图 → AI理解 → 生成操作步骤 → 前端Action协议执行 → 操作日志

| 组件 | 技术方案 | 说明 |
|------|--------|------|
| **A2UI** | AI驱动的操作界面 | 而非传统表单，大模型自动填表、推荐参数 |
| **FunctionCalling** | 大模型工具调用 | 调用后端定义的操作函数（审批、导出、调整等） |
| **前端Action协议** | JSON格式定义 | 统一的操作协议：操作类型+参数+验证规则 |
| **操作执行** | 后端API | 真实执行操作、记录日志、支持回滚 |
| **日志系统** | PostgreSQL + Timeline | 记录所有操作历史、支持详情查看和回滚 |

**典型操作流程**：
```
用户语句："审批所有金额<1000的报销"
    ↓
大模型理解：自动审批操作 + 条件参数 (amount < 1000)
    ↓
A2UI自动填表：操作模板选择 + 条件参数输入
    ↓
FunctionCalling：调用 approve_expense_batch() 函数
    ↓
前端Action协议验证 + 执行
    ↓
操作日志记录（可回滚）
```

### 4️⃣ **经营诊断** (FunctionCalling + 数仓/OLAP + 大模型分析)

**核心流程**：定义诊断指标 → 数仓查询 → AI分析 → 生成报告+建议

| 组件 | 技术方案 | 说明 |
|------|--------|------|
| **FunctionCalling** | 大模型工具调用 | 按需调用各类诊断分析函数 |
| **数仓/OLAP** | Clickhouse或PostgreSQL + 聚合表 | 快速多维分析，支持时间序列、对比分析 |
| **诊断引擎** | Python规则引擎 | 异常检测（比值、趋势、阈值）+ 根因分析 |
| **大模型分析** | 结构化提示词 | 基于数据+规则，生成自然语言诊断报告 |
| **报告生成** | Markdown + 图表 | 可视化的诊断报告，包含建议方案 |

**诊断维度**：
- **订单指标**：订单量、金额、转化率 → 异常识别 + 根因（竞品/季节/营销）
- **转化指标**：点击率、加购率、购买率 → 漏斗分析 + 优化建议
- **客户指标**：活跃度、复购率、流失率 → 客户分层 + 保留策略
- **财务指标**：毛利率、ROI、成本占比 → 成本优化 + 定价策略

## 🔄 跨模块技术协同

### 共同基础设施
```
┌─────────────────────────────────┐
│  大模型服务（豆包/通义千问API）  │
│  FunctionCalling + 流式输出      │
└─────────────────────────────────┘
         ↓         ↓         ↓        ↓
    知识问答   数据查询   智能操作   经营诊断
```

### 数据流协同
- **知识库 → 数据查询**：文档中的数据定义可用于SQL Schema生成
- **数据查询 → 经营诊断**：查询结果作为诊断数据源
- **经营诊断 → 智能操作**：诊断建议转化为可执行的操作
- **智能操作 → 知识库**：操作日志可被知识问答系统召回
