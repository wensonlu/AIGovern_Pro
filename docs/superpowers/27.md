Structured Output + Markdown 重构计划（整理版）
一、重构上下文与目标
1.1 核心问题
- 当前后端全量返回扁平Markdown字符串，格式不规范（列表嵌套、代码块、间距等），导致前端需反复修复CSS，维护成本持续上升。
- 前端渲染依赖后端输出质量，需额外做格式容错处理，逻辑繁琐。
1.2 重构意义
- 提升可靠性：后端统一保证输出结构，前端无需额外容错。
- 简化前端逻辑：渲染从“异常处理”转为“规范渲染”，降低维护成本。
- 长期复用：定义统一Schema，后续扩展无需重复调整格式。
1.3 目标状态（结构化输出示例）

二、核心决策（结合用户确认需求）
- LLM侧策略：采用「纯Prompt工程」，通过在Prompt中指定JSON格式，引导LLM输出结构化内容（稳定性80-90%，无需依赖Claude Beta API）。
- 流式处理策略：采用「逐块返回section」，LLM生成一个section就实时返回一个，前端同步更新，优化用户体验。
- 实施顺序：采用「RAG优先」，先改造RAG服务（知识问答最常用）作为试点，验证通过后再推进SQL、诊断等其他服务。
- 兼容性：旧API保持不变，新增结构化API（非流式+流式），前端自动检测格式，支持新旧版本并行运行。

# 核心模型片段
from enum import Enum
from typing import Union, Optional
from pydantic import BaseModel, Literal

class SectionType(str, Enum):
    TEXT = "text"
    LIST_ORDERED = "list_ordered"
    LIST_UNORDERED = "list_unordered"
    CODE_BLOCK = "code_block"
    TABLE = "table"

# 文本段落
class TextSection(BaseModel):
    type: Literal["text"]
    markdown: str  # 支持Markdown富文本

# 列表项（支持嵌套）
class OrderedListItem(BaseModel):
    title: str
    details_markdown: Optional[str] = None
    subitems: Optional[list["OrderedListItem"]] = None

# 有序列表section
class ListOrderedSection(BaseModel):
    type: Literal["list_ordered"]
    items: list[OrderedListItem]

# 其他section类型（无序、代码块、表格）省略...

# 联合类型，支持自动识别section类型
Section = Union[TextSection, ListOrderedSection, ListUnorderedSection, CodeBlockSection, TableSection]

# 完整结构化响应
class StructuredChatResponse(BaseModel):
    sections: list[Section]
    sources: list[SourceReference]
    confidence: float
    session_id: str
    timestamp: datetime
    intent: str
    workflow: list[WorkflowStep]

# 流式事件模型
class StructuredStreamEvent(BaseModel):
    type: str  # "sources" | "section" | "done" | "error"
Phase 2：后端服务改造（优先RAG，1.5小时）
2.1 RAG服务改造（核心试点）
文件：backend/app/services/rag_service.py
- 新增方法：async def stream_with_structure(...)，实现流式逐块返回section。
- 核心逻辑：检索文档→返回sources事件→构建Prompt（指定结构化格式）→流式调用LLM→逐块解析section并返回→返回done事件。
- 新增辅助方法：_build_prompt_for_structured_output（Prompt工程，指定JSON格式）、_parse_to_structured（解析LLM输出为结构化section）。
# 核心方法片段（stream_with_structure）
async def stream_with_structure(self, question: str, ...) -> AsyncIterator[dict]:
    # 1. 检索文档
    sources = await self._retrieve_documents(question)
    # 2. 返回sources事件
    yield {"type": "sources", "sources": sources, "confidence": confidence_score}
    # 3. 构建结构化Prompt（核心：指定JSON格式）
    prompt = self._build_prompt_for_structured_output(question, context)
    # 4. 流式调用LLM，逐块积累内容
    accumulated_content = ""
    async for chunk in self.llm_client.stream_text(prompt):
        accumulated_content += chunk
        # 5. 实时解析，生成section就返回（逐块返回核心逻辑）
        sections = self._parse_to_structured(accumulated_content, question)
        for section in sections:
            yield {"type": "section", "section": section.model_dump()}
    # 6. 返回完成事件
    yield {"type": "done", "confidence": confidence_score}
