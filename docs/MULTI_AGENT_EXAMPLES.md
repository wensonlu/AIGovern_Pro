# 多Agent协作系统 - 复杂意图处理

## 概述

系统支持识别和处理**复杂意图**（一个问题包含多个意图），通过多个Agent并行执行后自动合并结果。

---

识别的意图：

- 子意图: business_diagnosis (诊断是否正常), knowledge_qa (获取改进建议)

执行流程：
┌─────────────────────────────────────────────────────────┐

        │              │              │

   │数据查询  │  │诊断分析  │  │知识问答  │
   │Agent     │  │Agent     │  │Agent     │
   │          │  │          │  │          │
   │查询SQL:  │  │计算指标: │  │检索文档: │

   │orders    │  │          │  │          │

## 系统架构

### 多Agent处理流程

```
User Input
    ↓
[意图分析器]
    ↓
┌─ 单一意图? ─→ [传统流程] ─→ Response
│
└─ 复杂意图 ─→ [多Agent协调器]
                    ↓
           ┌────────┼────────┐
           ↓        ↓        ↓
       [Agent1] [Agent2] [Agent3]  (并行执行)
       (Query)  (Diagnosis) (RAG)
           ↓        ↓        ↓
           └────────┼────────┘
                    ↓
           [结果聚合器]
                    ↓
           [LLM合并逻辑]
                    ↓
             [Response]
```

### Agent执行结果结构

每个Agent的输出格式统一：

```python
{
    "type": "agent_type",  # data_query, business_diagnosis, knowledge_qa
    "data": {...},         # Agent特定的数据
    "count": 5,            # 结果数量
    "error": None          # 错误信息（如果有）
}
```

## 结果合并策略

### 1. 数据收集
- 收集所有Agent的输出
- 标准化格式
- 提取关键信息

### 2. 上下文构建
为LLM构建合并上下文：

```
【数据查询结果】
- SQL: SELECT ... FROM ...
- 数据行数: 1000
- 前3条示例: [{...}, {...}, {...}]

【诊断分析结果】
- 发现问题数: 3
- 核心指标: {"订单数": 800, "转化率": 2.5%}
- 具体问题: [{"issue": "转化率低", "severity": "high"}]

【知识库检索结果】
- 相关文档数: 5
- 文档标题: ["营销策略", "客户留存", ...]
```

### 3. LLM合并
使用LLM生成综合回答：

```prompt
用户问题: "..."

已执行的分析:
[上述上下文]

请基于上述所有分析结果，生成一个综合回答。要求：
1. 整合所有Agent的输出
2. 逻辑清晰，避免重复
3. 突出主要结论
4. 提供可行的建议
```

### 4. 置信度计算
```
置信度 = (Agent1_conf + Agent2_conf + Agent3_conf) / 3

- 数据查询: 0.95 (高，基于SQL结果)
- 诊断分析: 0.92 (中高，基于指标)
- 知识问答: min(0.90, 0.7 + doc_count/50) (动态)
```

### 5. Source合并
汇聚所有Agent的信息来源：

```
Sources:
✓ 数据库查询 (100% 置信度)
✓ 营销策略.md (85% 相关度)
✓ 客户留存.md (78% 相关度)
✓ 指标分析 (100% 置信度)
```

## 使用方法

### 更新agent_service.py

在 `process_message_structured_stream` 中添加：

```python
async def process_message_structured_stream(
    self, message: str, db: Session, ...
) -> AsyncIterator[dict[str, Any]]:
    # 1. 先尝试复杂意图分析
    from app.services.multi_agent_service import multi_agent_service
    
    analysis = await multi_agent_service.analyze_complex_intent(message)
    
    if analysis.is_complex:
        # 2. 使用多Agent处理
        response = await multi_agent_service.process_complex_message(
            message, db, session_id, top_k
        )
    else:
        # 3. 使用原流程
        response = await self.process_message(message, db, session_id)
```
