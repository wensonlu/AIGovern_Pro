# AI Agent 架构知识库：裸模型与 Harness

> 来源：Wenson Lu 整理 · 2026-04-14
> 背景：药厂 AI Engineer 面试题引发的深度思考

---

## 一、核心定义

**Agent** 是一个能够自主**感知环境、做出决策、执行行动并从结果中学习**的智能体。

四个关键动词：**感知 → 决策 → 执行 → 学习**

---

## 二、裸模型的四大硬伤

严格来说，裸模型只能做"决策"这一步——给定输入，产出输出。

| 硬伤 | 说明 |
|------|------|
| ❌ 无记忆 | 无法保留跨会话信息 |
| ❌ 不能执行代码 | 只能输出，无法自验证 |
| ❌ 知识过时 | 训练数据有截止日期 |
| ❌ 无工作环境 | 没有手脚，无法触达外部世界 |

---

## 三、Harness 六大组件

Harness 是大脑之外的一切——**感官系统、运动系统、记忆系统、能量系统**——负责让"想"变成"做"。

| # | 组件 | 解决的硬伤 | 核心作用 |
|---|------|-----------|----------|
| 1 | **文件系统** | 无记忆 | 统管存储与版本管理 |
| 2 | **Bash + 沙箱** | 不能执行代码 | 赋予代码自验证能力 |
| 3 | **AGENTS.md** | 知识过时 | 无需训练即可注入知识 |
| 4 | **Web Search + MCP** | 知识过时 | 打破知识截止限制 |
| 5 | **上下文工程** | 无工作环境 | 动态管理上下文，抗信息腐烂 |
| 6 | **编排 + Hooks** | 多 Agent 协同 | 保障多 Agent 协作质量 |

---

## 四、三层架构总结

```
┌─────────────────────────────────────────┐
│           Model = 大脑                   │
│         负责"想"——决策                  │
├─────────────────────────────────────────┤

│         负责"做"——感知、执行、学习      │
│                                         │

│   Web Search+MCP │ 上下文工程 │          │
│   编排+Hooks                            │
└─────────────────────────────────────────┘
```

## 五、底层支撑体系

1. **增强型上下文管理** — 动态调度、多源接入、垃圾回收（MCP 协议）
2. **运行时行为约束** — 权限隔离、状态管理、错误兜底（沙箱环境）
3. **全链路评估体系** — 单元/任务/安全评估的自动化体系

---

## 六、面试答题角度：Hermes vs OpenClaw

> 问题："用 1-2 句话说清楚 Hermes 优于 OpenClaw 之处"

**核心答案框架：**

> "裸模型只是脑子，Harness 是让 AI 长出手脚的那个框架。Hermes 在 Harness 基础上加了**学习循环**——执行→评估→提炼→检索，让记忆组件从被动存储变成**主动提炼**，用得越久 Agent 越懂你。"

**两者定位对比：**

| 维度 | Hermes | OpenClaw |
|------|--------|----------|
| 核心定位 | Agent 运行时 + 内置学习循环 | 多通道自动化 + 路由控制 |
| 记忆机制 | 持久记忆 + 结构化经验积累 | 偏重配置，不内置长期学习 |
| 哲学 | 适应性强，越用越聪明 | 控制力强，workspace 原生 |
| 擅长场景 | 长期运行的 Agent | 多渠道运营、自动化工作流 |

## 七、关键洞察

1. **Harness 本质是"AI 的操作系统"** — 不只是工具，是让 AI 能够行动的基础设施
2. **MCP 正在成为 USB 接口标准** — 2026年 Figma/Google Colab 都在接入，生态扩张迅速
3. **学习循环是 Hermes 的核心差异点** — 不是比 OpenClaw 更好，而是解决不同问题
4. **面试题没有标准答案** — 考的是对 AI Agent 架构的理解深度，不是背书
