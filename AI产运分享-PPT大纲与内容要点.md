### 【第5页：Step 1 - 需求分析：AI帮你头脑风暴】

**标题：** Step 1：需求分析

**核心方法：**
```
业务背景 → AI结构化拆解 → 需求优先级 → 技术评估
```

**左侧：方法说明**
```
AI做的是什么？
- Socratic追问：不断问"为什么"，挖到根因
- 结构化拆解：把模糊的想法变成具体的需求点
- 优先级判断：帮你理清先做哪个
```

**右侧：演示Prompt模板**
```
我是一个[公司类型]的产品经理
我们现在遇到了[问题描述]
用户群体是[用户画像]
目标[业务指标提升]

请帮我分析：
1. 这个问题背后真正的原因是什么？
2. 用户有哪些未被满足的需求？
3. 有哪些可能的解决方案？
4. 你会建议先做哪个？为什么？
```

**讲解要点（8分钟）：**
```
（3分钟讲解方法）
重点讲Socratic追问——像苏格拉底一样不断追问"为什么"

（5分钟现场演示）
拿一个真实业务需求，现场演示：
输入：运营同学的一个痛点
输出：结构化的需求文档

关键：让观众看到"AI是怎么追问的"
```

### 【第11页：Step 6 - MVP开发：Google Antigravity】

**标题：** Step 6：MVP开发

**工作流（横向流程图）：**
```
设计稿描述 → Antigravity理解 → 代码生成 → 自动测试 → Artifact汇报
                                ↓
                            你来审核
```

**Antigravity两大模式：**
```
┌────────────────────────────────────────────┐
│              Editor View                   │
│  传统AI辅助编程，适合需要你参与的场景       │
└────────────────────────────────────────────┘
                                           
┌────────────────────────────────────────────┐
│             Manager View                   │
│  Agent自主执行，适合复杂任务，你只管审核    │
└────────────────────────────────────────────┘
```

**MVP的核心价值：**
```
不是"做完"，是"验证"
用最小成本验证：用户是否真的需要这个功能
```

**讲解要点（7分钟）：**
```
（3分钟）讲清楚两个View的区别
Editor View：你写代码，AI在旁边辅助
Manager View：AI自主完成任务，你只管审核结果

（4分钟）讲MVP逻辑
重点：不是"做一个完整产品"
是"做一个最小版本，验证核心假设"

案例：
如果你的假设是"用户愿意付费"
MVP只需要：展示产品+付费按钮
不需要：完整支付流程、发票系统、客服体系
```

### 【第12页：Step 7 - 效果实验：让数据说话】

**标题：** Step 7：效果实验

**实验设计框架：**
```
┌─────────────────────────────────────────────────┐
│                                                 │
│   假设：用户看到X功能，会更愿意做Y行为           │
│                                                 │
│   验证：AB测试 / 灰度发布 / 问卷调研             │
│                                                 │
│   指标：点击率 / 转化率 / 留存率 / 完件率        │
│                                                 │
│   结论：做对了什么 / 做错了什么 / 下一步         │
│                                                 │
└─────────────────────────────────────────────────┘
```

**AI辅助环节：**
```
假设生成 → ChatGPT：基于功能生成假设
问卷设计 → ChatGPT：设计验证问卷
数据分析 → Perplexity：行业基准对比
结论报告 → ChatGPT：数据→洞察
```

**三个常见陷阱：**
```
❌ 只看短期数据 → 伤害长期留存
❌ 没设对照组 → 不知道是不是你的功劳
❌ 指标单一 → 某个涨了，其他跌了
```

**讲解要点（5分钟）：**
```
重点讲"为什么做完不算完"

案例：
功能A：上线后日活涨了10%
功能B：上线后转化率涨了5%，但留存跌了3%

问题：哪个功能是"真的好"？
答案：功能B可能是"短期刺激"，功能A更健康

核心观点：
做完不是目的，验证才是目的
```

   每个环节都要问：怎么证明它有效？

3. 工具会用还不够，要用透
   选3个工具用熟，比50个工具都试试强

串联起来才是真正的效率提升

选3个最适合自己的工具，先用起来

## ⚠️ 制作注意事项

### 时间把控
- 每页PPT只讲1个重点
- 演示环节留足Buffer（演示可能失败）
- Q&A留10分钟

### 避免的问题
- ❌ 字太多（每页不超过50字）
- ❌ 念稿（PPT是提示器，不是演讲稿）
- ❌ 技术术语堆砌（面向产运，要说人话）
- ❌ 什么都讲（时间不够，内容取舍）

### 现场准备清单
```
□ PPT提前拷贝到演示电脑
□ Stitch和Antigravity网址收藏好
□ 如果网络不好，提前截图备用
□ 录屏软件准备好（以防万一）
□ 激光笔或翻页器
□ 电脑充电器
```
